Run It

First, run beanstalkd on one or more machines. There is no configuration file and only a handful of command-line options. 

$ ./beanstalkd -l 10.0.1.5 -p 11300

This starts up beanstalkd listhing on address 10.0.1.5, port 11300. For more information on how to run beanstalkd as a background service in production, see the adm directory

Use It

Firt have one process put a job into the queue:

# Connect to pool
@beanstalk = Beaneater::Pool.new(['localhost:11300'])

# Enqueue jobs to tube
@tube = @beanstalk.tubes["my-tube"]
@tube.put '{ "key" : "foo" }', :pri => 5

Then start another process to take jobs out of the queue and run them:

# Process jobs from tube
while @tube.peek(:ready)
  job = @tube.reserve
  puts "job value is #{job.body["key"]}!"
  job.delete
end
